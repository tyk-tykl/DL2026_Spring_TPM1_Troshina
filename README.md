# DL2026_Spring_TPM1_Troshina

## Описание проекта
Данный репозиторий содержит выполнение задания по анализу и улучшению цифрового сервиса (сайт Белорусской железной дороги) с точки зрения:
- Business Analysis (BA)
- Product Management (PM)
- AI-инструментов

В работе были выявлены ключевые пользовательские проблемы, предложены решения и выбрана наиболее перспективная AI-инициатива.

## Структура репозитория

### 🔹 /ba — Business Analysis
- `users_and_functions.md` — основные функции сервиса и категории пользователей  
- `process_ticket.md` — описание процесса покупки билета + блок-схема  
- `problems.md` — ключевые пользовательские проблемы  

### 🔹 /pm — Product Management
- `vision.md` — продуктовое видение на 2 года  
- `goal.md` — стратегическая цель на 1 год  

### 🔹 /ai — AI-часть
- `ai_ideas.md` — сгенерированные AI-идеи  
- `best_idea.md` — выбор лучшей идеи и обоснование  
- `prompts.md` — использованные промпты  
- `reflection.md` — рефлексия использования AI  

### 🔹 /other — Дополнительные материалы
- исходный файл с выполненным заданием    

## Ключевые результаты анализа

### Основные пользователи
- студенты  
- командировочные  
- пенсионеры  
- туристы  
- семьи с детьми  
- жители пригородов  
- компании и грузоотправители  

### Основные функции сервиса
- поиск расписания поездов  
- просмотр цен  
- покупка билетов  
- ознакомление с правилами перевозки  
- получение информации об услугах  

### Ключевые проблемы

**1. Нестабильность сайта**  
Сайт может быть недоступен или работать медленно при высокой нагрузке.

**2. Сложный интерфейс**  
Пользователям трудно ориентироваться и оформлять билеты.

**3. Проблемы с авторизацией**  
Пользователи не могут войти в аккаунт или сталкиваются с ошибками.

## Предложенные решения

Были предложены AI-решения:
- AI-помощник выбора места  
- AI-чат для покупки билетов  
- AI-прогноз наличия билетов  
- AI-планировщик поездок  

## Выбранное решение

**AI-чат для покупки билетов**

### Причины выбора:
- упрощает пользовательский путь  
- снижает сложность интерфейса  
- подходит для широкой аудитории  
- может внедряться поэтапно  

## Продуктовая цель

Повысить стабильность и надёжность системы авторизации, чтобы пользователи могли без проблем входить в аккаунт и пользоваться сервисом.

## Итог

В ходе работы были:
- выявлены ключевые проблемы пользователей  
- предложены практические решения  
- выбрана реалистичная AI-инициатива  

Предложенные улучшения направлены на повышение удобства использования сервиса, снижение количества ошибок и увеличение конверсии покупки билетов.

## Использование AI

AI-инструменты использовались для:
- генерации идей  
- структурирования информации  
- анализа пользовательских проблем  

Финальные выводы, приоритизация и выбор решений были сделаны самостоятельно.
